ทำไมต้องทดสอบ ?

Single Page Application หรือ SPA เป็นแนวทางการทำเว็บไซต์อีกรูปหนึ่ง ที่เข้ามาแก้ปัญหาหลายๆอย่างในการทำเว็บไซต์

ลดปัญหา Server Load

SPA จะช่วยลดจำนวน HTTP Request ที่ browser ร้องขอไปยัง server เพราะทุกครั้งที่เปลี่ยนหน้าจะไม่มีการร้องขอไฟล์ static ต่างๆใหม่จะมีการเรียกแค่ไฟล์ที่จำเป็นต้องใช้เท่านั้น

ลดความซับซ้อนของโปรแกรมในฝั่ง server

การแสดงผลทั้งหมดจะถูก render ในฝั่ง client ทำให้ไม่มีความจำเป็นต้องใช้ template engine ในฝั่ง server อีกต่อไป
ลืมไปเลย Twig, Smarty, Jade, Blade
ประหยัด CPU ไม่เปลือง Memory ของ server ทำให้รองรับผู้ใช้ได้เพิ่มขึ้นอีก

แยก Frontend/Server-side Developer ได้ชัดเจน

Frontend เขียน SPA เป็นหลัก ส่วน Server-side ก็ทำงานกับ Database เป็นหลัก เขียน WebService เพื่อให้ฝั่ง Frontend เอาไปใช้ แถม WebService ที่เขียนยังเอาไปต่อยอดกับการทำ Mobile App ได้อีก

SEO คืออุปสรรค์

แต่สิ่งเดียวที่ไม่เป็นใจ ก็คือ SEO เพราะ Search Engine ไม่สามารถเข้าใจ Javascript ได้, แต่ก็มี Guidline ต่างๆของ Google ที่พยายามจะแก้ไขปัญหานี้ และนี่คือสิ่งที่เรากำลังจะทำการทดลองกัน

วัตถุดิบของเรา

- เว็บนี้เขียนขึ้นด้วย Angular JS แบบง่ายๆ เปิด HTML5 mode และไม่มีอะไรซับซ้อน
- ใช้ Prerender Service ในการทำ HTML Snapshot สำหรับ Crawler
- ใช้ Nginx เป็น WebServer
- resource ทั้งหมดเป็น Static Content ไม่มี Server Side Programmer นะคร้าบ
- deploy ผ่าน Docker
